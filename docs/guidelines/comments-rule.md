## Codex용 주석 규칙(권장 템플릿)

### 1) 주석은 “왜(why)”만

* 허용: 비즈니스 정책/제약, 성능 트레이드오프, 레거시 호환 워크어라운드, 보안/권한 경고, 비자명한 알고리즘 요지
* 금지: 코드가 그대로 말하는 내용(what/how) 반복

  * 예: `// set loading true`, `// assign value` 류 금지

### 2) 주석은 “희귀하게”, 복잡 블록 앞에 1~2줄

* 복잡한 조건 분기/상태 전이/대용량 테이블 최적화 같은 곳만
* 함수/변수명으로 의도가 드러나면 주석 대신 리팩터링 우선

### 3) TODO/FIXME는 티켓/만료 조건 필수

* 포맷 고정: `// TODO(PROJ-123): ... (remove when ...)`
* 티켓 없으면 TODO 금지(시간 지나면 쓰레기 주석이 됨)

### 4) “정책/스펙/IA”는 주석이 아니라 문서로

* 화면 전체 동작, 권한 매트릭스, 상태 정의표는 md 문서에 두고
* 코드에는 링크/요약 1줄만 남긴다

### 5) 자동 생성되는 “문서형 주석” 제한

* public API 타입/함수에만 짧은 doc 주석 허용
* 컴포넌트/훅에 장문의 주석 금지(구조로 해결)
