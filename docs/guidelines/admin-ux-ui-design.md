# TOPIK AI Admin UX/UI 원칙

## 문서 계약

- 기준 모듈: Dashboard, Users, Community, Message, Operation, Commerce, Assessment, Content, Analytics, System
- 메뉴명은 `Users`를 코드 식별자로 유지하되, 사용자 노출 라벨은 역할에 맞는 한글 명칭을 사용합니다.
- 감사 추적 용어는 `감사 로그`로 통일합니다.
- Users 상세 탭은 `프로필`, `활동`, `결제`, `커뮤니티`, `로그`, `관리자 메모` 6개를 고정합니다.

## 라벨 언어 규칙

- 사용자에게 노출되는 메뉴, 브레드크럼, 페이지 제목, 탭, 버튼, 상태 라벨은 기본 한글로 작성합니다.
- 동일 개념을 영어와 한글로 혼용하지 않습니다.
- 영문 표기 예외는 제품명, 외부 서비스명, 코드/식별자, 법적 고유 명칭처럼 한글 치환 시 의미가 손상되는 경우로 제한합니다.
- 코드 식별자와 URL 경로는 영문을 유지할 수 있지만, 화면 라벨은 별도 한글 매핑을 둡니다.

## 운영 기본 흐름

- 공통 흐름은 `검색 -> 상세 -> 조치 -> 감사 로그 확인`입니다.
- 콘텐츠형 화면은 `작성/수정 -> 확인 -> 발행 -> 감사 로그 확인` 흐름을 추가로 가집니다.
- 조치성 화면은 성공 피드백과 함께 `감사 로그 확인` 진입점을 제공해야 합니다.

## 관리자 페이지 유형

관리자 페이지는 아래 4가지 패턴으로 분류합니다. 새 페이지를 만들 때는 이 패턴 중 하나를 먼저 선택하고, 예외가 필요할 때만 변형합니다.

### 1) 목록 운영형

- 목적: 대량 데이터 조회, 필터링, 상태 조치
- 기본 구성:
  - `PageTitle`
  - 필요 시 요약 카드 3~4개
  - `Card(extra=선택적 주요 액션 버튼)`
  - `Card body` 첫 줄의 `SearchBar`
  - `SearchBar`는 `검색 대상 Select + 검색 Input + 상세 버튼 + 총 건수` 순서를 유지
- 상세 조건은 `상세` 버튼 클릭 시 열리는 드롭다운 패널로 분리하고, 페이지 기준 날짜 필드의 기간 범위만 배치합니다. 패널 하단에는 `필터 초기화`와 `적용` 버튼을 둡니다
- 드롭다운 안에서 바꾼 기간 값은 즉시 조회하지 않고 draft 상태로만 유지합니다. 실제 목록 조회는 `적용` 버튼 클릭 시점에만 발생합니다.
  - `상세 검색` 드롭다운에는 항상 `DatePicker.RangePicker` 기반 기간 범위 필터를 포함합니다
  - 메인 `Table`
  - 행 클릭 `Drawer` 또는 상세 모달
- 추천 컴포넌트:
  - `Card`, `Select`, `Input`, `Modal`, `Statistic`, `DatePicker`, `Table`, `Drawer`, `Tag`, `Popconfirm`
- 적합한 페이지:
  - 회원 목록
  - 강사 관리
  - 추천인 관리
  - 쿠폰 관리
  - 포인트 관리
  - 이커머스 관리

### 2) 정책/시나리오 편집형

- 목적: 정책, 규칙, 시나리오, 운영 설정 편집
- 기본 구성:
  - `PageTitle`
  - 상단 상태/버전 바
  - 좌측 목록 또는 상단 탭
  - 우측 편집 폼 또는 메인 편집 영역
  - 저장/게시/복제 액션 바
  - 테스트 또는 변경 이력 패널
- 추천 컴포넌트:
  - `Form`, `Tabs`, `Segmented`, `Card`, `Alert`, `Drawer`, `Modal`, `Timeline`
- 적합한 페이지:
  - 이벤트
  - 챗봇 설정
  - 학습 미션

### 3) 카탈로그/자산 관리형

- 목적: 자산 목록과 메타데이터 관리
- 기본 구성:
  - `PageTitle`
  - 유형/상태 중심 필터
  - 메인 `Table`
  - 우측 상세 `Drawer`
  - 상세 안 탭: 기본 정보, 메타데이터, 연결 자산, 변경 이력
- 추천 컴포넌트:
  - `Table`, `Drawer`, `Descriptions`, `Tabs`, `Tag`, `Badge`, `Empty`
- 적합한 페이지:
  - 문제은행
  - 콘텐츠 관리
  - 배지
  - 단어장

### 4) 편집/미리보기 병행형

- 목적: 콘텐츠/시험 구성과 결과 미리보기를 동시에 확인
- 기본 구성:
  - `PageTitle`
  - 상단 상태/버전/발행 여부
  - 좌측 편집 폼
  - 우측 미리보기 패널
  - 하단 저장/검수/발행 액션
- 추천 컴포넌트:
  - `Form`, `Card`, `Splitter` 또는 2열 레이아웃, `Tabs`, `Preview Panel`, `Affix` 액션 바
- 적합한 페이지:
  - EPS TOPIK
  - 레벨 테스트
  - 소나기
  - 객관식 선택

## 레이아웃 배치 규칙

- 목록형 페이지: `PageTitle -> Summary Cards(선택) -> Card(extra=선택적 주요 버튼, body=SearchBar -> 안내 문구 -> Table)`
- 우측 상단 버튼은 `등록/생성`, `내보내기`, `즉시 실행`처럼 해당 페이지의 핵심 액션이 분명할 때만 둡니다. 단순 페이지 이동 버튼은 안내 문구나 테이블 링크로 내립니다.
- `SearchBar` 바깥에는 검색과 직접 관련 없는 필터를 두지 않고, `상세 검색` 드롭다운에는 기간만 둡니다.
- `정렬`, `상태`, `카테고리`, `역할`, `수단`처럼 비기간 조건은 `상세 검색`에 넣지 않습니다.
- 동일한 목록형이라면 `검색 대상 -> 검색어 -> 상세 -> 총 건수` 위치를 페이지마다 바꾸지 않습니다. `메시지 > 메일/푸시`, `커뮤니티 > 게시글 관리/신고 관리`, `운영 > FAQ`는 동일한 카드 내부 목록 패턴을 기준으로 맞춥니다.
- 상세 drill-down: 링크 이동 또는 `Drawer`
- 편집형 페이지: `PageTitle -> Status Bar -> Editor -> Action Bar`
- placeholder 페이지: `PageTitle -> 정보 Alert -> 권한 안내 -> 예정 기능 리스트`

## 컴포넌트 선택 규칙

- 목록의 빠른 조치는 `Table` 행 단위 액션으로 제공합니다.
- 상세 확인은 전체 페이지 이동보다 `Drawer`를 우선 검토합니다.
- 작성/편집 맥락이 강한 화면은 `Modal`보다 별도 편집 영역 또는 전용 페이지를 우선 사용합니다.
- 정책 충돌이나 파괴적 액션은 `ConfirmAction` 또는 `Modal.confirm`으로 명시적으로 확인합니다.

## 상호작용 규칙

- 파괴적 액션은 확인 단계와 사유 입력 수단을 제공합니다.
- 성공 피드백은 대상 요약과 `감사 로그 확인` 경로를 함께 제공합니다.
- URL 파라미터로 목록, 필터, 탭 상태를 복원합니다.
- Drawer나 편집 패널을 닫아도 목록 필터와 선택 상태는 가능한 한 유지합니다.
