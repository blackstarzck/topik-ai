# TOPIK AI Admin 정보 구조(IA)

이 문서는 관리자 페이지 문서들의 기준 축입니다. 다른 문서의 메뉴명, 탭명, 로그 용어는 본 문서를 기준으로 통일합니다.

## 2) 문서 간 용어 통일 규칙

- 메뉴명은 `Users`(복수형)로 표기합니다. `User` 단수형은 사용하지 않습니다.
- 관리자 행동 기록은 `감사 로그`로 통일합니다.
- `시스템 로그`는 서버/API 기술 로그, `감사 로그`는 관리자 조치 로그로 구분합니다.
- `회원 상세` 탭은 `프로필 / 활동 / 결제 / 커뮤니티 / 로그 / 관리자 메모`로 고정합니다.
- 본 프로젝트 범위에서 `Learning`, `Content/Course` 메뉴는 제외합니다.

## 3) 문서 연결 관계

- `docs/specs/admin-page-analysis.md`: 위 IA의 각 모듈 목적과 핵심 기능 정의
- `docs/specs/admin-page-tables.md`: 위 IA 기준으로 페이지별 목록 컬럼/필터/액션 정의
- `docs/specs/admin-user-detail-page-structure.md`: `Users > 회원 상세` 화면의 상세 설계
- `docs/specs/admin-action-log.md`: `System > 감사 로그` 상세 설계

## 4) 운영 관점 원칙

- 운영 기본 흐름은 `검색 -> 상세 진입 -> 조치 -> 감사 로그 확인`입니다.
- 조치 가능한 핵심 화면은 `회원 목록`, `회원 상세`, `게시글 관리`, `신고 관리`, `메시지 > 메일`, `메시지 > 푸시`입니다.
- 데이터 변경 액션은 반드시 `감사 로그`에 남겨 추적 가능해야 합니다.
