# Codex 응답 완료 체크리스트 (재발 방지)

## 목적
- 작업 결과를 보고할 때, 미완료 항목/남은 작업 누락으로 인한 오해를 방지한다.

## 적용 범위
- Codex의 구현/리팩토링/버그수정/문서수정 작업 완료 보고 전반

## 필수 보고 형식 (완료 응답)
- `변경 요약`
- `변경 파일 목록`
- `요구사항 영향도 보고`
- `검증 결과(실행/미실행 포함)`
- `남은 작업`
- `리스크와 후속 제안`

## 요구사항 영향도 보고 규칙
- 사용자 요구사항이 프로젝트에 미친 영향 범위를 최소 `영향 모듈`, `데이터 계약`, `공통 UI`, `운영/정책`, `검증 범위` 기준으로 명시한다.
- 영향이 없는 항목도 `없음` 또는 `영향 없음`으로 적어 생략 때문에 판단이 흔들리지 않게 한다.
- 문서 수정/삭제가 요구사항에 포함된 경우 어떤 MD를 왜 바꿨는지 함께 적는다.

## 남은 작업 작성 규칙
- 남은 작업이 없으면 반드시 `없음`으로 명시한다.
- 남은 작업이 있으면 항목별로 아래 3가지를 함께 적는다.
  - 미완료 이유
  - 영향 범위
  - 다음 실행 액션

## 금지 규칙
- 남은 작업 섹션을 생략한 채 완료처럼 보이게 보고하지 않는다.
- 검증 미실행/실패를 숨기지 않는다.
- 후속 조치가 필요한데 `완료`로만 단정하지 않는다.

## 셀프 게이트 (응답 직전 확인)
- `완료/미완료 경계`가 명확한가?
- `남은 작업`을 명시했는가?
- `검증 상태`를 사실대로 작성했는가?
- 사용자가 즉시 다음 결정을 내릴 정보가 포함됐는가?
