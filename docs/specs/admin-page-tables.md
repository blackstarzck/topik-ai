# TOPIK AI Admin 페이지별 테이블/필드 정의

이 문서는 관리자 페이지의 `테이블 컬럼`, `입력 필드`, `필터`, `정렬`, `액션`을 정리합니다.  
페이지 목적은 `admin-page-analysis.md`, 화면 블록과 미확정 정책은 `docs/specs/page-ia/*.md`를 먼저 참고합니다.

## 공통 규칙

- 연결 가능한 식별자(ID, 작성자, 결제 번호)는 링크 스타일 텍스트를 우선 사용합니다.
- 링크 대상이 없는 상세 정보는 Drawer 또는 모달로 확인합니다.
- placeholder 페이지는 실제 컬럼을 확정하지 않고 `예정 데이터 블록` 수준으로만 기록합니다.
- 파괴적 액션은 확인 단계와 사유 입력 수단을 함께 설계합니다.
- 목록 운영형 페이지는 `Card(extra=선택적 주요 버튼)` 내부에 `SearchBar -> 안내 문구(선택) -> Table` 순서를 유지합니다.
- `SearchBar`는 `검색 대상 선택 Select + 검색 Input + 상세 버튼 + 총 건수` 구조를 기본값으로 사용합니다.
- 상세 필터와 `필터 초기화`/`적용` 버튼은 `SearchBar`의 `상세 검색` 드롭다운 안에서 같은 위치 규칙으로 유지합니다.
- `상세 검색`에는 페이지 기준 날짜 필드의 기간만 두고, `정렬`, `상태`, `카테고리`, `역할`, `수단` 같은 비기간 항목은 넣지 않습니다.
- `상세 검색` 드롭다운에는 페이지 기준 날짜 필드를 대상으로 한 `DatePicker.RangePicker` 기간 필터를 항상 포함합니다.
- 드롭다운 안의 기간 값 변경은 draft로만 유지하고, 테이블 재조회와 URL 갱신은 `적용` 버튼 클릭 시점에만 수행합니다.
- 단순한 관련 화면 이동은 우측 상단 버튼보다 안내 문구 링크나 테이블 셀 링크를 우선 사용합니다.

## 1) 회원 관리 > 회원 목록

- 테이블 컬럼: 회원 ID, 이메일, 닉네임, 가입일, 최근 접속, 회원 상태, 등급, 구독 상태
- 필터: 검색어, 가입 기간
- 정렬: 가입일, 최근 접속, 회원 상태
- 액션: 회원 상세, 회원 정지/해제, 관리자 메모
- 레이아웃 메모: 요약 없는 목록 운영형 기준으로 `Card 내부 SearchBar -> Table` 순서를 유지합니다.

## 2) 회원 관리 > 강사 관리

- 현재 상태: 구현됨
- 구현 컬럼: 강사 ID, 이름, 이메일, 소속, 담당 국가, 계정 상태, 활동 상태, 담당 과정 수, 담당 학습자 수, 최근 활동, 최근 조치일
- 구현 필터: 검색어(강사명/강사 ID/이메일/대상 그룹), 최근 활동일
- 구현 액션: `대상 그룹 보기`, `강사 정지`, `강사 정지 해제`
- 상세 Drawer: 기본 정보, 전문 분야, 담당 과정, 최근 메시지 발송 이력, 관리자 메모, 감사 로그 확인(`Instructor`)
- 예정 데이터 블록:
  - 강사 기본 정보: 강사 ID, 이름, 소속, 상태
  - 운영 정보: 담당 과정 수, 담당 학생 수, 최근 활동, 계약/정산 상태
  - 액션: 상태 변경, 담당 과정 연결, 메시지 발송

## 3) 회원 관리 > 추천인 관리

- 현재 상태: placeholder
- 상세 IA 문서: `docs/specs/page-ia/users-referrals-page-ia.md`
- 예정 데이터 블록:
  - 추천 코드 정보: 코드, 추천인 회원, 코드 상태, 생성일, 만료일
  - 실적 정보: 피추천인 수, 추천 확정 수, 누적 보상, 최근 사용일, 이상치 여부
  - 상세 Drawer: 추천 관계 목록, 보상 원장, 정책 스냅샷, 운영 메모
  - 액션: 상세 보기, 코드 비활성화, 보상 조정, 추천 관계 확인, 감사 로그 확인
- 정책 메모:
  - 추천 확정 시점, 보상 수단, 수동 보정 권한이 아직 미확정이므로 실제 컬럼/상태값 확정 시 상세 IA 문서를 기준으로 함께 갱신합니다.

- 자동 발송 컬럼: 템플릿 ID, 카테고리, 템플릿명, 제목, 발송 그룹, 트리거, 최근 발송, 상태
- 수동 발송 컬럼: 템플릿 ID, 카테고리, 템플릿명, 제목, 발송 그룹, 최근 수정, 상태
- 입력 필드: 카테고리, 템플릿명, 설명, 제목, 발송 그룹, 트리거, HTML 본문, JSON 본문
- 액션: 생성, 수정, 삭제, 미리보기, 나에게 보내기, 즉시 발송, 예약 발송, 활성화/비활성화

- 입력 필드: 그룹명, 설명, 적용 채널, 정의 방식, 운영 상태, 세그먼트 조건, 정적 대상 목록

- 필터: 채널, 발송 방식(자동/수동), 검색어, 발송일

- 액션: 권한 관리 이동, 감사 로그 이동, 상세 보기

## 29) 시스템 > 권한 관리

- 테이블 컬럼: 관리자 ID, 이름, 역할, 권한 템플릿, 최근 변경일
- 필터: 역할, 검색어
- 정렬: 최근 변경일, 역할
- 액션: 역할 변경, 권한 부여, 권한 회수

## 30) 시스템 > 감사 로그

- 테이블 컬럼: 로그 ID, 대상 유형, 대상 ID, 조치, 수행자, 사유, 시각
- 요약 카드: 현재 결과 수, 권한 변경 로그 수, 오늘 생성 로그 수
- 필터: 검색어, 시각
- 정렬: 시각, 대상 유형
- 액션: 상세 보기, 원본 페이지 이동, 필터 초기화

## 31) 시스템 > 시스템 로그

- 테이블 컬럼: 로그 ID, 레벨, 컴포넌트, 메시지, 발생 시각
- 요약 카드: 오류 로그 수, 경고 로그 수, 영향 컴포넌트 수
- 필터: 검색어, 발생 시각
- 정렬: 발생 시각, 레벨
- 액션: 상세 보기, 컴포넌트 기준 관련 운영 화면 이동, 감사 로그 이동
