# TOPIK AI Admin 페이지별 테이블 설계

본 문서는 `docs/architecture/admin-information-architecture.md` 기준 메뉴를 그대로 따릅니다.

## 공통 상세 인터랙션 규칙

- 식별자(ID/작성자 등)로 기존 페이지 이동이 가능한 경우, 해당 셀은 링크 스타일 텍스트로 이동 동선을 제공합니다.
- 위 조건이 아닌 경우, 행 클릭 시 `Descriptions` 기반 상세 모달을 표시해 더미 상세 데이터를 확인합니다.
- 예외: `Users > 회원 목록`은 기존 행 클릭 상세 진입 패턴을 유지합니다.

## 공통 필터/정렬 규칙

- 모든 테이블은 액션 컬럼을 제외한 주요 데이터 컬럼에 `AntD Table`의 컬럼 드롭다운 `filters`와 `sorter`를 제공합니다.
- 컬럼 필터는 `filterSearch`를 기본 적용해 옵션 수가 늘어나도 운영자가 빠르게 좁힐 수 있어야 합니다.
- 외부 검색/상태 필터가 있는 화면도 컬럼 단위 필터/정렬을 함께 제공하되, 기존 운영 흐름(`검색 -> 상세 -> 조치 -> 감사 로그 확인`)을 깨지 않아야 합니다.

- 변경 전/후 Role 및 권한 목록을 감사 로그에서 비교 가능해야 한다.
- `Target Type=Admin`, `Target ID=<관리자 ID>` 기준으로 감사 로그 역추적이 가능해야 한다.

## 13) System > 감사 로그

감사 로그 상세는 `docs/specs/admin-action-log.md`를 기준으로 관리합니다.

### 필수 연결 규칙
- Users/Community/Operation/Billing에서 발생한 데이터 변경 액션은 감사 로그에 기록
- 감사 로그에서 대상 ID를 통해 원본 화면으로 역추적 가능해야 함

### 필터/정렬
- 컬럼 드롭다운: 로그 ID / 대상 유형 / 대상 ID / 조치 / 수행자 / 사유/근거 / 시각
