# TOPIK AI Admin 페이지별 테이블 설계

본 문서는 `docs/architecture/admin-information-architecture.md` 기준 메뉴를 따릅니다.

## 공통 규칙

- 링크 가능한 식별자(ID/작성자 등)는 링크 스타일 텍스트를 우선 사용합니다.
- 그 외 상세는 행 클릭 후 `Descriptions` 기반 모달 또는 Drawer로 확인합니다.
- placeholder 페이지는 테이블 대신 역할/예정 기능 요약 블록을 우선 배치합니다.

## 1) Users > 회원 목록
- 컬럼: User ID, 이메일, 닉네임, 가입일, 최근 접속, 회원 상태, 등급, 구독 상태
- 액션: 회원 상세, 회원 정지/해제, 관리자 메모

## 2) Users > B2B 그룹 관리
- 현재 상태: placeholder
- 예정 핵심 필드: 그룹명, 유형, 소속 회원 수, 담당자, 상태, 최근 수정

## 3) Users > 추천인 관리
- 현재 상태: placeholder
- 예정 핵심 필드: 추천인 코드, 추천자, 피추천 회원 수, 보상 상태, 최근 수정

## 4) Community > 게시글 관리
- 컬럼: 게시글 ID, 제목, 작성자, 게시판, 작성일, 조회수, 댓글수, 신고수, 상태
- 액션: 상세, 숨김, 삭제

## 5) Community > 신고 관리
- 컬럼: 신고 ID, 게시글, 대상 사용자 ID, 신고자, 신고 사유, 신고일, 처리 상태
- 액션: 게시글 숨김, 사용자 정지, 처리 완료

## 6) Message > 메일 / 푸시
- 자동 발송 컬럼: 템플릿 ID, 카테고리, 템플릿명, 제목, 발송 그룹, 자동 조건, 최근 발송, 상태
- 수동 발송 컬럼: 템플릿 ID, 카테고리, 템플릿명, 제목, 발송 그룹, 최근 수정, 상태
- 편집 필드: 카테고리, 템플릿명, 요약, 제목, 발송 그룹, 자동 조건, HTML 본문, JSON 본문

## 7) Message > 대상 그룹
- 컬럼: 그룹명, 설명, 정의 방식, 조건 요약, 예상 대상 수, 상태, 작업
- 입력 필드: 그룹명, 설명, 정의 방식, 적용 채널, 운영 상태, 설정 방식, 세그먼트 조건, 정적 대상 목록
- 액션: 저장, 조회하기, 재계산, 삭제

## 8) Message > 발송 이력
- 컬럼: 템플릿명, 발송 그룹, 수신자 수, 발송 성공, 발송 실패, 상태, 실행 시각
- 상세 Drawer: 기본 정보, 수신자 샘플 테이블, 검색, 재시도, 복제 발송
