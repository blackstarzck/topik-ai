# Users > 회원 상세 페이지 구조

이 문서는 `docs/architecture/admin-information-architecture.md`의 `Users > 회원 상세` 구조를 상세화한 문서입니다.

## 화면 개요

```text
회원 기본 정보 카드
- 이메일 / 닉네임 / 가입일 / 상태
- 최근 로그인 / 회원 등급 / 구독 상태
- 액션: [계정 정지] [정지 해제] [탈퇴 처리]

Tabs
프로필 | 활동 | 결제 | 커뮤니티 | 로그 | 관리자 메모
```

## 1) 프로필

### 목적
- 회원의 기본 계정 정보를 확인하고 상태를 관리

### 표시 정보
| 항목 | 내용 |
| --- | --- |
| User ID | 회원 ID |
| Email | 로그인 이메일 |
| Nickname | 닉네임 |
| 가입일 | 회원 가입 날짜 |
| 최근 로그인 | 마지막 로그인 |
| 회원 상태 | 정상 / 정지 / 탈퇴 |
| 회원 등급 | 일반 / 프리미엄 |
| 구독 상태 | 구독 여부 |

### 관리자 액션
- 회원 정지
- 회원 정지 해제
- 회원 탈퇴 처리
- 회원 등급 변경
- 비밀번호 초기화

## 2) 활동

### 목적
- 사용자의 서비스 이용 패턴 확인

### 테이블 컬럼
| 컬럼 | 설명 |
| --- | --- |
| 활동 ID | 활동 번호 |
| 활동 유형 | 로그인 / 게시글 / 댓글 |
| 콘텐츠 | 대상 콘텐츠 |
| 활동 시간 | 활동 시간 |
| IP | 접속 IP |

### 필터
- 활동 유형
- 기간

### 정렬
- 활동 ID / 활동 유형 / 콘텐츠 / 활동 시간 / IP

## 3) 결제

### 목적
- 회원 단위 결제/환불 이력을 조회하고 Billing과 연결

### 테이블 컬럼
| 컬럼 | 설명 |
| --- | --- |
| 결제 ID | 결제 번호 |
| 상품 | 상품명 |
| 결제 금액 | 결제 금액 |
| 결제 수단 | 카드 / 계좌 |
| 결제일 | 결제 날짜 |
| 상태 | 완료 / 취소 / 환불 |

### 액션
- 결제 ID 클릭 시 Billing > 결제 내역으로 이동

### 필터/정렬
- 결제 ID / 상품 / 결제 금액 / 결제 수단 / 결제일 / 상태

## 4) 커뮤니티

### 테이블 컬럼
| 컬럼 | 설명 |
| --- | --- |
| 게시글 ID | 게시글 번호 |
| 제목 | 게시글 제목 |
| 게시판 | 게시판 |
| 작성일 | 작성 날짜 |
| 신고수 | 신고 수 |
| 상태 | 정상 / 숨김 |

### 액션
- 게시글 상세보기
- 게시글 숨김
- 게시글 삭제

### 필터/정렬
- 게시글 ID / 제목 / 게시판 / 작성일 / 신고수 / 상태

## 5) 로그

### 목적
- 보안 및 문제 추적

### 테이블 컬럼
| 컬럼 | 설명 |
| --- | --- |
| 로그 ID | 번호 |
| 로그 유형 | 로그인 / API |
| IP | 접속 IP |
| Device | 디바이스 |
| 시간 | 로그 시간 |

### 필터/정렬
- 로그 ID / 로그 유형 / IP / Device / 시간

## 6) 관리자 메모

### 목적
- CS 기록 및 운영 히스토리 관리

### 테이블 컬럼
| 컬럼 | 설명 |
| --- | --- |
| 메모 ID | 번호 |
| 관리자 | 작성 관리자 |
| 내용 | 메모 내용 |
| 작성일 | 작성 날짜 |

### 액션
- 메모 작성
- 메모 수정
- 메모 삭제

### 필터/정렬
- 메모 ID / 관리자 / 내용 / 작성일

## 운영 흐름

```text
1. 회원 검색
2. 회원 상세 진입
3. 문제 확인
4. 조치
5. 감사 로그에서 조치 내역 확인
```

### 예시
- 회원 신고 -> User Detail -> 커뮤니티 탭 -> 게시글 숨김
- 결제 문의 -> 결제 탭 -> 결제 확인 -> Billing 상세 이동
- 계정 문제 -> 로그 탭 -> 로그인 기록 확인
