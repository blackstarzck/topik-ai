# TOPIK AI Admin 페이지 분석

이 문서는 관리자 메뉴별 `목적`, `책임 범위`, `핵심 기능`, `페이지 유형`을 상위 수준에서 정리합니다.  
세부 화면 블록은 `admin-page-ia-drafts.md`, 실제 컬럼과 입력 필드는 `admin-page-tables.md`를 기준으로 봅니다.

## 문서 사용 규칙

- `admin-page-analysis.md`: 왜 이 페이지가 존재하는지, 어떤 책임을 가지는지 정리합니다.
- `admin-page-ia-drafts.md`: 페이지 레이아웃, 섹션 순서, Drawer/Modal, 추천 컴포넌트를 정리합니다.
- `admin-page-tables.md`: 테이블 컬럼, 입력 필드, 필터, 정렬, 액션을 정리합니다.
- placeholder 페이지는 이 문서와 `admin-page-ia-drafts.md`를 먼저 채우고, 실제 UI 설계가 고정되면 `admin-page-tables.md`를 확정합니다.

## 모듈별 분석

## 1) 대시보드

- 목적: 운영 상태와 위험 신호를 첫 화면에서 빠르게 파악
- 책임 범위: KPI 요약, 작업 큐, 주요 알림, 최근 조치 요약
- 핵심 기능: 운영 현황 카드, 빠른 진입, 처리 대기 큐, 위험 알림, 관련 모듈 이동
- 페이지 유형: 대시보드형

## 2) 회원 관리

- 목적: 회원 운영과 인접한 운영 데이터를 한 축에서 관리
- 책임 범위: 회원 목록, 회원 상세, 강사 관리, 추천인 관리
- 핵심 기능:
  - 회원 검색과 상세 확인
  - 상태 변경과 관리자 메모
  - 분리된 강사 운영 데이터 관리
  - 추천 코드와 보상 정책 운영
- 페이지 유형:
  - 회원 목록: 목록 운영형
  - 회원 상세: 상세 드릴다운형
  - 강사 관리: 목록 운영형
  - 추천인 관리: 목록 운영형

## 3) 커뮤니티

- 목적: 게시글 운영과 신고 처리 흐름 관리
- 책임 범위: 게시글 관리, 신고 관리
- 핵심 기능:
  - 게시글 숨김/삭제
  - 신고 검토와 처리
  - 회원/게시글 상세로 연계
- 페이지 유형: 목록 운영형

## 4) 메시지

- 목적: 메일과 푸시를 공용 대상 그룹 기준으로 운영
- 책임 범위: 메일, 푸시, 대상 그룹, 발송 이력
- 핵심 기능:
  - 자동/수동 발송
  - 템플릿 관리
  - 대상 그룹 조회와 저장
  - 발송 결과 추적과 재시도
- 페이지 유형:
  - 메일/푸시: 정책 편집형 + 목록 운영형 혼합
  - 대상 그룹: 목록 운영형
  - 발송 이력: 목록 운영형

## 5) 운영

- 목적: 서비스 운영성 콘텐츠와 정책 화면을 관리
- 책임 범위: 공지사항, FAQ, 이벤트, 챗봇 설정
- 핵심 기능:
  - 운영 콘텐츠 게시와 종료
  - FAQ 카테고리와 공개 상태 관리
  - FAQ 검색/상세 확인/공개 여부 전환
  - 이벤트 정책과 노출 운영
  - 챗봇 시나리오와 인계 규칙 관리
- 페이지 유형:
  - 공지사항, FAQ: 목록 운영형
  - 이벤트: 정책/시나리오 편집형
  - 챗봇 설정: 정책/시나리오 편집형

## 6) 커머스

- 목적: 결제와 판매 정책을 한 축에서 운영
- 책임 범위: 결제 내역, 환불 관리, 쿠폰 관리, 포인트 관리, 이커머스 관리
- 핵심 기능:
  - 결제/환불 조회와 처리
  - 결제/환불 동일 원본 기반 상태 동기화
  - 회원 상세, 감사 로그, 환불 처리 화면 간 역추적
  - 쿠폰/포인트 정책 관리
  - 상품/패키지 판매 상태 관리
- 페이지 유형:
  - 결제 내역, 환불 관리: 목록 운영형
  - 쿠폰 관리, 포인트 관리, 이커머스 관리: 목록 운영형

## 7) 평가

- 목적: 시험과 문항 운영을 위한 별도 작업 공간 제공
- 책임 범위: 문제은행, EPS TOPIK, 레벨 테스트
- 핵심 기능:
  - 문항 자산 관리
  - 시험 세트 구성
  - 평가 규칙과 결과 기준 운영
- 페이지 유형:
  - 문제은행: 카탈로그/자산 관리형
  - EPS TOPIK: 편집/미리보기 병행형
  - 레벨 테스트: 편집/미리보기 병행형
- 현재 상태: placeholder 중심. 상세 IA는 `admin-page-ia-drafts.md` 우선

## 8) 콘텐츠

- 목적: 학습 콘텐츠와 보상형 콘텐츠 운영 기준 정리
- 책임 범위: 콘텐츠 관리, 배지, 단어장, 소나기, 객관식 선택, 학습 미션
- 핵심 기능:
  - 콘텐츠 메타데이터 관리
  - 콘텐츠 유형별 편집
  - 보상/미션 규칙 연결
- 페이지 유형:
  - 콘텐츠 관리, 배지, 단어장: 카탈로그/자산 관리형
  - 소나기, 객관식 선택: 편집/미리보기 병행형
  - 학습 미션: 정책/시나리오 편집형
- 현재 상태: placeholder 중심. 상세 IA는 `admin-page-ia-drafts.md` 우선

## 9) 통계

- 목적: 운영 의사결정을 위한 핵심 지표 확인
- 책임 범위: 통계 개요
- 핵심 기능:
  - 사용자, 메시지, 커뮤니티, 매출 지표 요약
  - 기간 필터 기반 교차 분석
  - 다른 운영 모듈과 교차 분석 진입
  - 커머스 원본 데이터와 연결된 매출/환불 지표 확인
- 페이지 유형: 대시보드형

## 10) 시스템

- 목적: 관리자 계정, 권한, 로그 체계를 운영
- 책임 범위: 관리자 계정, 권한 관리, 감사 로그, 시스템 로그
- 핵심 기능:
  - 관리자 계정 필터링과 상태 파악
  - 관리자 권한 제어
  - 조치 이력 추적과 원본 페이지 역추적
  - 기술 로그 확인과 컴포넌트별 영향 화면 연결
- 페이지 유형: 목록 운영형
