# TOPIK AI Admin 페이지 분석

본 문서는 `docs/architecture/admin-information-architecture.md`의 IA를 기준으로, 각 모듈의 목적과 핵심 기능을 정리합니다.

## 1) Dashboard

### 목적
- 운영 현황을 한 화면에서 파악하고 우선 처리 항목을 빠르게 식별

### 핵심 기능
- 핵심 KPI 카드(가입자, 활성 사용자, 신고 건수, 발송 성공률)
- Work Queue(미처리 신고/환불/문의)
- 장애 및 실패 경고 알림(Alerts)

## 2) Users

### 목적
- 회원 조회, 상태 제어, 이슈 대응을 단일 흐름으로 처리

### 핵심 기능
- 회원 검색 및 목록 조회
- 회원 상세 탭 기반 분석(프로필/활동/결제/커뮤니티/로그/관리자 메모)
- 계정 상태 조치(정지, 정지 해제, 탈퇴 처리)

## 3) Community

### 목적
- 게시판 운영 품질 유지 및 신고 대응 자동화

### 핵심 기능
- 게시글/게시판/카테고리 관리
- 신고 접수, 검토, 처리 상태 관리
- 게시글 숨김/삭제 및 작성자 제재 연계

## 4) Message

### 목적
- 메일과 푸시를 공용 대상 그룹 기준으로 운영하고, 채널별 이력을 분리 관리

### 핵심 기능
- 메일/푸시 각각 자동 발송과 수동 발송 관리
- 공용 대상 그룹 관리(정적 그룹/조건 기반 그룹, Drawer 기반 세그먼트 편집, 예상 대상 수 계산)
- 채널별 발송 이력, 수신자 샘플 Drill-down, 실패 건 재시도, 복제 발송

## 5) Operation

### 목적
- 서비스 운영 공지성 콘텐츠를 일관되게 관리

### 핵심 기능
- 공지사항 CRUD
- FAQ 관리
- 배너/이벤트 노출 관리

## 6) Billing

### 목적
- 결제/환불 운영의 정확성 확보

### 핵심 기능
- 결제 내역 조회
- 환불 처리 및 상태 추적
- 회원 상세(결제 탭)와 양방향 이동

## 7) Analytics

### 목적
- 운영 의사결정을 위한 지표 추적

### 핵심 기능
- 사용자/커뮤니티/메시지/매출 지표 대시보드
- 기간 비교 및 추이 분석
- 운영 액션 효과 측정(예: 메시지 발송 후 리텐션 변화)

## 8) System

### 목적
- 관리자 권한, 정책, 로그를 통한 거버넌스 확보

### 핵심 기능
- 관리자 계정/역할/권한 관리
- 정책 관리(이용약관/개인정보/운영 정책)
- 감사 로그 조회
- 시스템 로그(API/서버) 모니터링

## 모듈 간 연결 포인트

- Users <-> Billing: 회원 상세 결제 탭에서 결제 내역으로 이동
- Community -> System: 게시글 숨김/삭제, 사용자 제재는 감사 로그로 추적
- Message -> Analytics: 채널별 발송 성공률, 전환율 지표 연동
- Operation/Community/Users의 변경 액션은 모두 System의 감사 로그에 기록
