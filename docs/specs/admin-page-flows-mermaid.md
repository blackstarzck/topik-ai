# TOPIK AI Admin 페이지 흐름도

## 개요

- 공통 흐름은 `검색/필터 -> 상세/확인 -> 조치 -> 피드백 -> 감사 로그 확인`입니다.
- placeholder 모듈은 `메뉴 진입 -> 역할/예정 기능 확인 -> 상세 정의 대기` 흐름을 가집니다.

## Users > 회원 목록

```mermaid
flowchart LR
  A["회원 목록 진입"] --> B["검색/필터 적용"]
  B --> C["결과 테이블 확인"]
  C -->|"행 클릭"| D["회원 상세 이동"]
  C -->|"조치 실행"| E["확인 모달"]
  E -->|"확인"| F["상태 변경"]
  F --> G["감사 로그 확인"]
```

## Users > 강사 관리

```mermaid
flowchart LR
  A["강사 관리 진입"] --> B["검색/필터 적용"]
  B --> C["강사 목록 확인"]
  C -->|"행 클릭"| D["강사 상세 Drawer"]
  D --> E["담당 과정/메시지 이력 확인"]
  D --> I["대상 그룹 보기"]
  D -->|"정지/해제"| F["확인 모달 + 사유 입력"]
  F --> G["상태 변경"]
  G --> H["감사 로그 확인 (Instructor)"]
```

## Message > 메일 / 푸시

```mermaid
flowchart LR
  A["채널 진입"] --> B["자동/수동 탭 선택"]
  B --> C["목록 확인 또는 등록 버튼 클릭"]
  C -->|"템플릿 등록"| D["템플릿 등록 모달(메타 정보)"]
  D -->|"등록"| E["테이블 갱신"]
  E -->|"행 클릭 또는 등록 상세 이동"| F["등록 상세 페이지"]
  C -->|"템플릿 정보 수정"| G["템플릿 정보 수정 모달"]
  G --> E
  E -->|"미리보기/테스트"| I["미리보기/테스트"]
  F --> H["TinyMCE 본문 작성 및 저장"]
  H --> I
  I --> J["발송 실행"]
  J --> K["발송 이력 확인"]
  K --> L["감사 로그 확인"]
```

## Operation > 공지사항

```mermaid
flowchart LR
  A["공지사항 진입"] --> B["목록 확인"]
  B -->|"행 클릭"| C["공지 미리보기 Modal"]
  B -->|"공지 등록"| D["공지 등록 상세 페이지"]

```

## Operation > 이벤트

```mermaid
flowchart LR
  A["이벤트 진입"] --> B["SearchBar 검색 및 기간 필터"]
  B --> C["목록 확인"]
  C -->|"행 클릭"| D["이벤트 상세 Drawer"]
  C -->|"이벤트 등록"| E["이벤트 등록 상세 페이지"]
  D -->|"수정"| F["이벤트 수정 상세 페이지"]
  D -->|"게시 예약 또는 종료"| G["확인 모달 + 사유 입력"]
  D -->|"참여 현황 확인"| H["참여 현황 패널 또는 관련 화면"]
  E --> I["기본 정보, 노출 설정, 참여 조건, 보상 설정 작성"]
  I --> J["선택형 SEO/공유 메타 입력"]
  F --> I
  J --> K["저장 또는 게시 예약"]
  K --> L["목록 복귀 + URL 상태 복원"]
  G --> M["감사 로그 확인"]
  L --> D
```
