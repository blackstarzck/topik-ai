# TOPIK AI Admin 페이지별 상세 IA 초안

이 문서는 페이지별 상세 정보구조 초안을 정리합니다.  
`admin-page-analysis.md`가 페이지의 목적과 책임을 설명하고, `admin-page-tables.md`가 컬럼과 필드를 정의한다면, 이 문서는 그 사이에서 `화면 블록`, `주요 액션`, `추천 컴포넌트`, `상세 진입 방식`, `연관 화면`을 설계합니다.

## 작성 규칙

- 상태 표기:
  - `구현됨`: 실제 화면이 동작하는 상태
  - `placeholder`: 메뉴 자리와 권한 경계만 확보된 상태
  - `기획 필요`: 세부 정책과 데이터 계약이 아직 없는 상태
- 화면 구성은 `상단 요약 -> 필터/검색 -> 본문 -> 상세/후속 액션` 순서로 적습니다.
- B2C 영향이 있는 페이지는 마지막에 노출 위치 또는 사용자 영향 포인트를 적습니다.

## 페이지 유형 분류

### 1) 목록 운영형

- 주 용도: 대량 데이터 검색, 상태 비교, 빠른 조치
- 기본 레이아웃:
  - `PageTitle`
  - 요약 카드 3~4개
  - `FilterBar`
  - `Table`
  - 행 클릭 `Drawer`
  - 우측 상단 주요 액션 버튼
- 추천 컴포넌트:
  - `Card`, `Statistic`, `Input.Search`, `Select`, `DatePicker`, `Table`, `Drawer`, `Tag`, `Badge`

### 2) 정책/시나리오 편집형

- 주 용도: 운영 규칙, 정책, 시나리오 편집
- 기본 레이아웃:
  - `PageTitle`
  - 상태/버전 바
  - 좌측 목록 또는 상단 탭
  - 메인 편집 영역
  - 하단 저장/게시/복제 액션
  - 테스트 또는 이력 패널
- 추천 컴포넌트:
  - `Form`, `Tabs`, `Segmented`, `Alert`, `Card`, `Modal`, `Timeline`

### 3) 카탈로그/자산 관리형

- 주 용도: 자산 목록과 메타데이터 운영
- 기본 레이아웃:
  - `PageTitle`
  - 유형/상태 중심 필터
  - `Table`
  - 우측 `Drawer`
  - Drawer 안 탭: 기본 정보, 메타데이터, 연결 자산, 변경 이력
- 추천 컴포넌트:
  - `Table`, `Drawer`, `Descriptions`, `Tabs`, `Tag`, `Badge`, `Timeline`

### 4) 편집/미리보기 병행형

- 주 용도: 시험/콘텐츠 구성과 결과 미리보기 동시 확인
- 기본 레이아웃:
  - `PageTitle`
  - 상태/버전/발행 바
  - 좌측 편집 폼
  - 우측 미리보기 패널
  - 하단 저장/검수/발행 액션 바
- 추천 컴포넌트:
  - `Form`, `Card`, `Tabs`, 2열 레이아웃, `Affix` 액션 바, 미리보기 패널

## 상세 IA 초안

## 1) 회원 관리 > 강사 관리

- 현재 상태: placeholder
- 페이지 유형: 목록 운영형
- 목적: 강사 목록과 강사 운영 상태를 관리
- 주요 role: `OPS_ADMIN`, `SUPER_ADMIN`
- 진입 경로: `회원 관리 > 강사 관리`
- 화면 구성:
  - 상단 요약: 전체 강사 수, 활성 강사 수, 휴면 강사 수
  - 필터 바: 강사명, 강사 ID, 소속, 상태, 담당 국가
  - 본문 테이블: 강사 기본 정보, 담당 과정 수, 담당 학생 수, 최근 활동
  - 상세 Drawer: 강사 프로필, 소속 정보, 담당 과정, 메시지 발송 이력
- 핵심 기능:
  - 강사 검색과 상세 확인
  - 상태 변경
  - 담당 과정/학생 연결 확인
  - 강사 대상 공지 발송 연계
- 추천 컴포넌트: `Statistic`, `Table`, `Drawer`, `Tag`, `Select`
- 상세 진입 방식: 테이블 행 클릭 Drawer
- 연관 페이지: `회원 상세`, `메시지 > 대상 그룹`, `운영 > 이벤트`
- B2C 영향: 강사 정보, 담당 과정 노출, 강사 대상 공지 발송
