# TOPIK AI Admin 관리 데이터 - B2C 노출 맵

이 문서는 관리자 페이지에서 다루는 테이블/정적 데이터/운영 입력 데이터가 B2C 화면 어디에 노출되고 어떻게 사용되는지 추적하기 위한 기준 문서입니다.

## 1) 목적

- 관리자 데이터 변경이 B2C 어디에 영향을 주는지 빠르게 판단
- 운영/기획/개발/QA가 같은 데이터 기준으로 협업
- placeholder 상태의 메뉴도 향후 B2C 연결 지점을 미리 기록

## 2) 기록 규칙

- 관리자 화면에서 생성, 수정, 삭제, 상태 변경이 가능한 데이터는 모두 이 문서의 대상입니다.
- B2C 실제 코드나 기획서로 확인되지 않은 노출 위치는 추정으로 표시합니다.
- 한 데이터가 여러 B2C 화면에 쓰이면 대표 노출 위치와 보조 노출 위치를 함께 적습니다.
- 내부 운영용 데이터는 `내부 전용`으로 명시합니다.

## 3) 노출 상태 정의

- `확인됨`: 현재 기준으로 B2C 노출 위치가 구현 또는 합의 문서로 확인됨
- `운영상 추정`: 관리자 도메인과 기능상 연결은 명확하지만, B2C 실제 구현/문서 확인은 아직 없음
- `내부 전용`: 관리자/배치/감사 목적으로만 사용되고 B2C 직접 노출이 없음
- `노출 예정`: 메뉴 자리는 확보했지만 B2C 노출 구조가 아직 정의되지 않음

## 5) 운영 메모

- 이 문서는 관리자 프론트엔드 저장소 기준으로 작성되며, B2C 실제 구현 저장소가 없으면 `운영상 추정`과 `노출 예정`을 사용합니다.
- `운영상 추정` 상태의 항목은 B2C 팀과 합의가 끝나면 `확인됨`으로 승격합니다.
- 새 테이블 또는 새 정책 데이터가 생기면 같은 작업에서 이 문서에 반드시 추가합니다.
